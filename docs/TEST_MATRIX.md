# Test Matrix

Behavior-to-proof control panel cho validation expectations.

## Format

| Behavior | Unit | Integration | E2E | Platform | Evidence |
|----------|------|-------------|-----|----------|----------|
| Example: User login flow | ✅ | ✅ | ✅ | N/A | tests/auth/login.test.ts |

## Legend

- ✅ — Passed
- ⏳ — In progress
- ❌ — Failed
- ⚠️ — Weak proof
- N/A — Not applicable
